Getting Started

Help Generator is designed to generate source-code documentation for programs written in Object Pascal.  Delphi, Kylix, and Lazarus are all used to create programs with Object Pascal.  If you are writing components, or open-source programs with one of those IDE's, Help Generator can help you create documentation for your project.  It is also useful for create documentation for your program for your own internal use.

As you will see, Help Generator is not intended to provide documentation for users of closed-source programs.  If that is what you want to do, there are many other programs you can use for that purpose.

To use Help Generator, you need to insert comments in the interface section of your source code files.  Generally the comments must immediately precede the item that they document.  Help Generator extracts those comments and creates the documentation from them.  The documentation can be either a series of web pages, an html-help project or a LaTeX file.  With a Latex file you can create files in other formats such as PDF, or Rich-Text format (rtf).  (See TeXnicCenter and Latex2Rtf.)

In your source-code comments, you can use a number of special tags to create cross-links or to control the appearance of the generated documentation.  The supported tags are documented at http://pasdoc.sipsolutions.net/SupportedTags.

Once you have put the comments in your source code, you are ready to run to Help Generator.  The most important things you need to do are to

  1. choose the source code files to include, and
  2. choose the output type.

Once you have done that, you are ready to go to the Generate tab, and generate the documentation.  However, there are a few more things that you might want to do to customize your documentation.  The remainder of this page describes those options.

You can save your settings for a project by selecting "File|Save".  Later on you can retrieve the projecting settings again by selecting "File|Open".

Source Files Tab

Enter the names of the source code files that you want Help Generator to process here or click the Browse button and select them.

Options Tab

Project Name

The Project Name is used to help determine the name of the output file or files.  For example if the project name is "MyDoc" and the output type is LaTeX, the LaTeX file that is generated would be MyDoc.tex.

Language

There are a number of places where Help Generator writes predefined phrases such as "All Units" in the documentation.  Those phrases have been translated into a variety of languages.  To ensure that the language for those phrases matches the language used in the comments, choose the correct language

Properties, Methods, Events, and Fields to Include

These check boxes controls which properties, methods, events, and fields to include based on their visibility.  For example, if you are documenting a component, you might want to include only the published and public properties because those are the ones to which the users of the component will have access.  However, it would still be easy to generate another version of the documentation for your own use that included the protected and private properties, methods, events, and fields as well.

"Implicit" means the fields at the beginning of a class declaration before any visibility specifier.  "Implicit" fields will be either public or published depending on the {$M+} or {$M-} compiler option.

Title

The Title is the title that will appear on the title page of the LaTeX documentation or as the caption of the web browser


Message Level

When Help Generator is creating the documentation, a number of informative messages are generated.  Some are error messages or warnings.  Others are just informative messages.  When the Message Level is set to 1, only errors and warning messages will be displayed.  When the Message Level is set to 2, a message will be shown for each source code file that is processed as well as errors and warnings.  As the Message Level increases to 6, the number of informative messages increases.

Output Type

The Output Type determines whether the generated documentation will be a series of web pages, an html-help project, an ordinary LaTeX file or a LaTeX for use with Latex2Rtf.

Locations Tab

Output Directory

The Output Directory is the location where the documentation should be created.  If you don't specify the location, the directory containing the first source code file you select will be chosen as the location to create the documentation.  If you specify a directory that does not exist, Help Generator will try to create it for you.

Introduction

If you want to include an introduction to your documentation, write it in a text file and specify its location here. Help Generator can process it in much the same way that it does source code files.

Conclusion

If you want to include a conclusion to your documentation, write it in a text file and specify its location here. Help Generator can process it in much the same way that it does source code files

Browser Location

This is the location of the web browser that you prefer to use for viewing web pages generated by Help Generator.  If left blank, the default browser for your computer will be used.

Pasdoc Consol Location

This is the location of the consol program pasdoc.exe.  Pasdoc.exe is included with Help Generator.  It is Pasdoc that actually creates the documentation for your source code files.

Html Help Contents File

If you are creating a Html Help file, you can specy the Html Help Contents File to use with the Html Help Project.  If you don't specify it, Help Generator will create a default one for you.

Web Options Tab

Header

If you are creating a web page as your output, the "Header" text will appear at the top of the web page.

Footer

If you are creating a web page as your output, the "Footer" text will appear at the bottom of the web page.

Latex Options

Line Breaks

If you are generating LaTeX output, you can use this option to help control the way line breaks are generated.  If you choose the "strict" option, TeX (the formatting engine for LaTeX files)  may extend a line past the normal page margin in order to prevent too many spaces from appearing between words.  If you choose the "sloppy" option, Tex will prevent words from extending past the page margin even if to do so it must insert too many spaces between words.

Hyph-en-at-ed Words

If you are generating LaTeX output, you can use this option to help control the way line breaks are generated.  Usually the reason LaTeX has problems processing a line is that it contains a long word near the end that it doesn't know how to hypenate.  You can tell it how to hypenate words by listing them here.  Put hyphens in the words wherever a hypen is allowed to appear.

GraphViz Options Tab

Create and Use GraphVizUses Diagram

GraphViz is a a set of open-source programs for creating diagrams.  Help Generator can create an input file for the "dot" program in GraphViz that shows how Object Pascal units use each other.  On Windows, Help Generator will run the "dot" program for you if you choose this option. On Linux, you must run dot yourself from the command line.

Note: the images can get very BIG so you may need to instruct users to set up their web browser's preferences so that it does not automatically resize the figure.  Also sometimes the image that is generated might be too big to be displayed properly in your web browser.  If so, the image may appear blank.

You can get GraphViz from http://www.graphviz.org/.

Create and use GraphViz Classes diagram

GraphViz is a a set of open-source programs for creating diagrams.  Help Generator can create an input file for the "dot" program in GraphViz that shows the class hierarchy in your program.   On Windows, Help Generator will run the "dot" program for you if you choose this option. On Linux, you must run dot yourself from the command line.

Note: the images can get very BIG so you may need to instruct users to set up their web browser's preferences so that it does not automatically resize the figure.  Also sometimes the image that is generated might be too big to be displayed properly in your web browser.  If so, the image may appear blank.

You can get GraphViz from http://www.graphviz.org/.

Location of GraphViz "Dot" Program

If you chose either of the previous two options, you must tell Help Generator where the "Dot" program is installed so Help Generator can run Dot for you.

Note: the images can get very BIG so you may need to instruct users to set up their web browser's preferences so that it does not automatically resize the figure.  Also sometimes the image that is generated might be too big to be displayed properly in your web browser.  If so, the image may appear blank.

You can get GraphViz from http://www.graphviz.org/.

LaTeX Graphics Package

If you are generating LaTeX output and use graphics in it (such as the graphics generated by GraphViz), you need to specify the LaTeX graphics package to use.  If you are planning to convert your LaTeX file to a PDF, use the PDF option.  If you are planning to convert it to a DVI file, use the DVI option.  If you are planning to convert your LaTeX file to some other format, specify None and manually edit your LaTeX to include the correct graphics package.

Required program: GraphViz

This edit box gives the URL for GraphViz.  You will need to install GraphViz if you wish to use the GraphViz Uses or Classes diagrams.

Spell Checking Tab

Check Spelling

Checking this check box causes spell-checking to be activated.  Spell checking is done by the consol program ASpell.

Language

Normally, you don't need to specify the language because it is automatically selected for you.  However, if the automatic selection is incorrect, specify the correct language here.

Words to Ignore

Frequently you will have words that ASpell doesn't recognize as correct even though they are correct.  Specify words that you want ASpell to ignore here.  Use one word per line.  (Note, when you generate documentation with spell checking activated, a list of all the misspelled words will be listed at the bottom of the messages.  You can copy that list here if all the words in it are spelled correctly.)

Required Program: ASpell

ASpell is the program that actually does the spell checking for PasDoc.  See http://pasdoc.sipsolutions.net/SpellChecking for more information.

Include Directories Tab

If any of your source files include other files via the include directive {$I AFile}, list the directories that have the include files here or click the Brows button to select them.

Defines Tab

If any of your files uses compiler directives such as {$IFDEF AConstant} to control compilation, list the constants that are defined here.  Help Generator will use them to help parse your source code files.

Generate Tab

Click the Generate Documentation button to generate the documentation for your project.  Messages will appear in the memo field above the button as the documentation is generated.  If spell checking is activated, a list of misspelled words will appear at the end of the messages.
